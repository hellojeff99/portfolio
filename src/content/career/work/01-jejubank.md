## 핵심 성과

| 결과 | 기여 |
| --- | --- |
| **약 1,000개 UI 이관** | 차세대 표준 가이드에 맞춰 레이아웃과 JavaScript 로직을 계정계 환경으로 전환 |
| **이관 데이터 정합성 확보** | AS-IS·TO-BE 화면 데이터를 비교하고 불일치 원인을 분석해 수정 |
| **여신 서비스 안정화** | JavaScript, MCI, C 서버 로직, Oracle SQL까지 전 구간을 추적해 오류 발생 지점을 식별 |

## AS-IS → TO-BE: 정보계 화면을 계정계로 이관

> **기존 정보계 여신 화면 약 1,000개를 차세대 표준에 맞는 계정계 화면으로 전환하고, 이관 전·후 조회 결과가 같은지 검증했습니다.**

| 구분 | 이관 작업 | 완료 기준 |
| --- | --- | --- |
| 화면 | 차세대 표준 가이드에 맞춰 레이아웃 전환 | 계정계 화면에서 정상 동작 |
| 로직 | JavaScript 오류 수정 및 중복 코드·반복문 개선 | 기존 기능과 동일하게 동작 |
| 인터페이스 | 화면별 송수신 데이터 포맷을 MCI에 등록 | 단말과 서버 간 데이터 송수신 확인 |
| 데이터 | AS-IS·TO-BE 화면의 조회 결과 비교 | 이관 전·후 데이터 일치 |
| 대외 연동 | 완료 화면은 실제 연동, 미완료 화면은 DB 임시 데이터로 테스트 | 연동 상태와 관계없이 화면 검증 |

## 데이터 불일치 해결

화면 오류를 UI 문제로 한정하지 않고, 데이터 흐름 전체를 단계별로 좁혀 원인을 찾았습니다.

```text-flow
오류 재현 → 테스트 데이터 확인 → JS 검토 → 서버 로그 추적 → SQL 검증 → 원인 공유, 수정, 재검증
```

### 해결 결과

- 데이터가 달라지는 지점을 화면·인터페이스·서버·DB 구간 중 하나로 특정
- 서버 로그와 C 로직의 실행 흐름을 확인해 잘못된 분기 가능성 검토
- 원장 조회 SQL을 분석해 조회 조건과 데이터 처리 오류 수정
- 분석 내용을 관련 담당자에게 공유하고 해결 방법을 검토한 뒤 재검증

## 오류 유형별 점검 항목


| 구간 | 점검 항목 |
| --- | --- |
| 화면 | JavaScript 코드, 입력값, 이벤트 처리 |
| 인터페이스 | 단말과 서버가 주고받는 MCI 데이터 포맷 |
| 서버 | 로그, C 로직의 분기와 데이터 처리 과정 |
| DB | 테스트 데이터 존재 여부, Oracle SQL과 조회 조건 |
